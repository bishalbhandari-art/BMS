# Book Management System (BMS)

The Book Management System (BMS) that allow users to add, edit, and display books. This includes features like age calculation of books, validation checks, and a genre filter. User can find their favorite books.

# Features

Enter book details such as Title, Author,ISBN, Publication Date, and Genre. Validates all input fields.

# 1.Requirement Phase

1. Requirements Phase Objective: Identify the features needed for the system. Example: Add, update, remove, and search books using the title or author name

2. Design Phase Objective: Plan how the system will be structured and operate. Example: Architecture: 3-tier (Ul, Business Logic, Database). Database: Tables for Books, Users, Borrowing Records

3. Implementation Phase Objective: Develop the system according to the design. Example: Develop features such as Add Book, Search Book, and Issue Book.

4. Testing Phase Objective: Verify the system works as expected. Example: Unit Test: Validate the Add Book module

5. Deployment Phase Objective: Release the system for users. Example: Host the application on a web server or a cloud platform such as AWS

6. Maintenance Phase Objective: Keep the system running smoothly and make necessary updates. Example: Fix bugs reported by users (eg, search not returning correct results).
